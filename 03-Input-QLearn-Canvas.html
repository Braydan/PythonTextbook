Learn how to collect user input to make your programs interactive and responsive.

What is Input?

Input is how your program receives information from the user. While output lets your program "speak" to users, input lets users "speak" to your program.

In Python, we use the input() function to get information from users. This makes your programs interactive - they can respond differently based on what the user types!

üí° Why is Input Important?

Without input, every program would do exactly the same thing every time. Input allows you to create personalized experiences, get user choices, and make programs that adapt to different situations!

The input() Function

Basic Usage

The input() function shows a message to the user and waits for them to type something:

name = input("What is your name? ")
print("Hello,", name)

How it works:

  1. The program displays the message in the parentheses
  2. The program waits for the user to type something and press Enter
  3. Whatever the user typed gets stored in the variable

‚úì Try It Yourself

Create a new file called greeting.py. Copy the code above and run it. Type your name when prompted and see what happens!

Storing Input in Variables

The input you collect needs to be stored somewhere so you can use it later. We store it in variables: 

age = input("How old are you? ")
print("You are", age, "years old")

Using Multiple Inputs

You can ask for multiple pieces of information:

first_name = input("First name: ")
last_name = input("Last name: ")
age = input("Age: ")

print("Hello,", first_name, last_name)
print("You are", age, "years old")

üìù Variable Naming Tips

  • Use descriptive names: age instead of a
  • Use underscores for multiple words: first_name
  • Lowercase letters for variable names
  • No spaces allowed in variable names

The Input is Always a String!

This is very important: Everything you get from input() is treated as text (a string), even if the user types a number!

The Problem

# This won't work as expected!
age = input("How old are you? ")
next_year = age + 1  # ERROR!

‚ö†Ô∏è Why does this fail?

Python thinks age is text, not a number. You can't add 1 to text! This will give you a TypeError.

Converting Input to Numbers

To use input as a number, you must convert it:

# For whole numbers (integers)
age = int(input("How old are you? "))
next_year = age + 1
print("Next year you'll be", next_year)

# For decimal numbers (floats)
height = float(input("Height in meters: "))
doubled = height * 2
print("Double your height is", doubled)

üí° Conversion Functions

  • int() - Converts to whole number (integer)
  • float() - Converts to decimal number (float)
  • str() - Converts to text (string) - rarely needed for input

Working with Different Data Types

Example: Calculator Program

# Get two numbers from user
num1 = float(input("Enter first number: "))
num2 = float(input("Enter second number: "))

# Perform calculations
sum_result = num1 + num2
difference = num1 - num2
product = num1 * num2
quotient = num1 / num2

# Display results
print("Sum:", sum_result)
print("Difference:", difference)
print("Product:", product)
print("Quotient:", quotient)

Example: Name and Age Program

name = input("What's your name? ")
age = int(input("How old are you? "))

print("Hello", name + "!")
print("In 10 years, you'll be", age + 10, "years old")
print("In 20 years, you'll be", age + 20, "years old")

‚úì Notice the Difference

For name, we keep it as a string (no conversion). For age, we convert to integer with int() so we can do math with it!

Input Validation Basics

Sometimes users type things that don't make sense for your program. Here are some ways to handle this:

Adding Clear Instructions

# Be specific in your prompt
age = int(input("Enter your age (numbers only): "))
choice = input("Would you like to continue? (yes/no): ")

What Happens with Bad Input?

# If user types "hello" instead of a number:
age = int(input("How old are you? "))
# This will cause a ValueError!

‚ö†Ô∏è Common Input Errors

If a user types letters when you're expecting a number, your program will crash with a ValueError. We'll learn how to handle these errors properly in the Error Handling lesson!

üí° Good Practice

Always give clear instructions in your input() prompts so users know exactly what type of information you're expecting.

Common Mistakes and How to Fix Them

Mistake 1: Forgetting to Store Input

# ‚ùå Wrong - input is displayed but not saved
input("What is your name? ")
print("Hello,", name)  # ERROR: name is not defined!

# ‚úÖ Correct - store input in a variable
name = input("What is your name? ")
print("Hello,", name)

Mistake 2: Forgetting to Convert to Number

# ‚ùå Wrong - can't do math with strings
age = input("How old are you? ")
next_year = age + 1  # ERROR!

# ‚úÖ Correct - convert to integer first
age = int(input("How old are you? "))
next_year = age + 1

Mistake 3: Missing Parentheses

# ‚ùå Wrong - missing closing parenthesis
name = input("What is your name? "

# ‚úÖ Correct - both parentheses present
name = input("What is your name? ")

Mistake 4: Converting When Not Needed

# ‚ùå Unnecessary - name should stay as string
name = int(input("What is your name? "))

# ‚úÖ Correct - keep name as string
name = input("What is your name? ")

Key Takeaways

  • input() is used to get information from users
  • Always store input in a variable to use it later
  • Input is always a string - convert it if you need a number
  • Use int() for whole numbers and float() for decimals
  • Give clear instructions in your input prompts
  • Make your prompts specific about what type of data you expect
  • Remember: Input makes your programs interactive and personalized!
